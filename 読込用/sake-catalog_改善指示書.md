# sake-catalog 改善指示書

対象: https://sake-catalog.vercel.app (Next.js / Vercel)
目的: 「自分に合う日本酒が見つかる → その場で購入できる」体験を完成させ、アフィリエイト収益でサイト運用費を賄う。
作成日: 2026-07-09

---

## 0. 前提と現状の課題

- 現状は単一ルート(`/`)のみで、探し方4種(地図・好み・銘柄・診断)と詳細表示がすべてモーダルで完結している。
- 銘柄ごとのURLが存在しないため、検索エンジンにインデックスされず、主要流入源である「銘柄名検索」からの流入がゼロ。
- 直近7日間のアクセスは5リクエスト。収益化の前に「インデックスされるページ構造」への転換が最優先。

**基本方針: モーダルUIの操作感は残しつつ、すべての銘柄・分類に固有URLを持たせる。**

---

## 1. KPI(優先順)

1. 銘柄詳細ページのGoogleインデックス数
2. 検索流入セッション数
3. 購入ボタンクリック率(詳細ページ閲覧→アフィリエイトクリック)
4. 導線別クリック数(診断/地図/好み/銘柄/ペアリング)

## 2. URL・ページ構造(サイトマップ)

```
/                          トップ
/sake/[slug]               銘柄詳細(収益の中核)★最優先
/brewery/[slug]            酒蔵詳細
/area/[prefecture]         都道府県別一覧(地図から探すの遷移先)
/type/[kaori|sou|jun|juku] 香味4分類別一覧(好みから探すの遷移先)
/search                    横断検索(銘柄名・条件フィルタ)
/diagnosis                 診断(質問フロー)
/diagnosis/result/[typeId] 診断結果(共有可能なURLにする)
/pairing/[category]        料理から逆引き(刺身/焼鳥/チーズ/鍋 など)
/scene/[scene]             シーン別(贈答/晩酌/初心者/パーティー)
/ranking                   人気・注目ランキング
/furusato                  ふるさと納税でもらえる日本酒特集(高単価導線)
/stats                     フレーバー統計レポート(被リンク獲得用資産)
/sommelier                 AI酒ソムリエ(対話型レコメンド、Phase 3)
/guide/[slug]              読み物・初心者ガイド(SEO記事)
/about                     サイトについて・運営者情報
/disclosure                広告掲載ポリシー(アフィリエイト表記)
/privacy                   プライバシーポリシー
/sitemap.xml, /robots.txt  自動生成
/llms.txt                  AI検索(LLMO)対策: サイト構成と主要ページの記述
```

- slugはローマ字推奨(例: `/sake/dassai-junmai-daiginjo-23`)。日本語URLでも可だが共有時のエンコードが崩れるためローマ字+日本語はメタで担保。
- 全ページ SSG または ISR(revalidate: 86400 目安)。クライアントレンダリング依存を排除する。

## 3. 銘柄詳細ページ `/sake/[slug]` の仕様 ★最重要

### 3-1. 表示コンテンツ(上から順)

1. **基本情報**: 銘柄名(ふりがな)/ 酒蔵名(リンク)/ 都道府県 / 特定名称(純米大吟醸など)
2. **味わいビジュアル**
   - 甘辛 × 淡麗濃醇 の2軸マップ上の位置
   - 香味4分類(薫・爽・醇・熟)のバッジ+一言説明
   - レーダーチャート(甘み/酸味/旨み/香り/キレ の5軸)
3. **スペック表**: 精米歩合 / 使用米 / 酵母 / アルコール度 / 日本酒度 / 酸度 / 火入れ(生酒等)
4. **おすすめの飲み方**: 温度帯(雪冷え〜熱燗)を帯グラフで適温表示
5. **ペアリング**: 合う料理3〜5件(それぞれ `/pairing/` へリンク)
6. **紹介文**: 200〜400字のオリジナル解説(コピー禁止・SEOの生命線)
7. **購入セクション** ※詳細は §6
8. **類似銘柄**: 「この銘柄が好きなら」レコメンド4件(味データの近傍で算出)
9. **酒蔵情報の要約**: 蔵の歴史一行+ `/brewery/` へリンク

### 3-2. SEO要件

- title: `【銘柄名】の味わい・スペック・購入先|サイト名`
- meta description: 味の特徴+ベネフィットを120字で
- 構造化データ: `Product`(offers付き)+ `BreadcrumbList`
- OGP画像: 銘柄名+味マップを描画した動的OG画像(`next/og` で生成)

## 4. 探し方の拡充

既存4種は維持し、以下を追加する。

| 導線 | 内容 | 優先度 |
|---|---|---|
| 料理から探す | 「今夜の料理に合う酒」逆引き。刺身/肉/揚げ物/チーズ/鍋 等 | 高 |
| シーンから探す | 贈答・初心者向け・晩酌デイリー・特別な日 | 高 |
| 価格帯で探す | 〜1,500円 / 〜3,000円 / 〜5,000円 / 5,000円〜 | 中 |
| 似ている酒から探す | 「獺祭が好き」→ 味データ近傍の銘柄提示 | 中 |
| スペックで探す | 日本酒度・精米歩合・生酒などの絞り込み(上級者向け) | 低 |

- 「好みから探す」の薫・爽・醇・熟には必ず一言説明を添える(例: 薫=フルーティで華やか)。初心者は用語を知らない前提で設計。
- トップは「迷ったら診断」を最も大きく配置し、その下に他導線を並べる。

---

## 5. 診断フローの仕様 `/diagnosis`

- 質問は5〜7問、選択式、所要1分以内。例:
  1. 普段よく飲むお酒は?(ワイン/ビール/焼酎/あまり飲まない…)
  2. 好きな果物の系統は?(柑橘/りんご・梨/トロピカル…)
  3. 甘口と辛口どちらが好み?
  4. 一緒に食べたい料理は?
  5. 冷やす派? 温める派?
- 結果は8〜12タイプに分類し、`/diagnosis/result/[typeId]` で固有URL化(SNS共有を想定)。
- 各タイプに「シェアしたくなるキャラ付きタイプ名」を命名し、タイプ名+味覚チャート入りの動的OG画像と「Xでシェア」ボタンを必須要件とする(広告費ゼロの主要流入源として設計)。
- 結果ページの構成: タイプ名+説明 → おすすめ銘柄3件(購入ボタン付き)→「もっと見る」で該当タイプ一覧へ。
- 診断結果ページは購入CVRが最も高くなる想定。専門EC・サブスク案件(§6-2)への誘導もここに置く。

## 6. アフィリエイト実装仕様

### 6-1. 購入セクションのUI(銘柄詳細内)

- 「この日本酒を購入する」見出し直下に **楽天市場 / Amazon / Yahoo!ショッピング** の3ボタンを横並び(モバイルは縦積み)。
- 返礼品が存在する銘柄には4つ目のボタン「ふるさと納税でもらう」を追加(報酬単価が物販の数倍。楽天ふるさと納税なら既存の楽天アフィリエイトIDで対応可)。
- 可能な限り個別商品への直リンク。無い場合のみ「銘柄名+容量」の検索結果リンクにフォールバック。
- 容量違い(720ml / 1800ml)はタブで切替。
- セクション冒頭に小さく「※本サイトはアフィリエイト広告を利用しています」。

### 6-2. 提携先

| 提携先 | 用途 |
|---|---|
| 楽天アフィリエイト | 物販メイン。日本酒の品揃えが最も広い |
| Amazonアソシエイト | 物販サブ。Prime需要 |
| バリューコマース等(Yahoo!) | 物販サブ |
| A8.net 等のASP | KURAND・saketaku等の日本酒専門EC/サブスク。高単価。診断結果ページに配置 |

### 6-3. リンク管理とクリック計測

- アフィリエイトURLをコンポーネントに直書きしない。`affiliate_links` データとして一元管理(sakeId × モール × URL)。
- 出口リンクは `/api/out/[sakeId]/[mall]` のリダイレクトAPI経由にし、サーバー側でクリックログ(sakeId, mall, 参照元導線, 日時)を記録してから302。
- `rel="sponsored nofollow noopener"` を必ず付与。
- GA4イベント `affiliate_click` を併用(パラメータ: sake_id, mall, source_flow)。

## 7. トップページの改修

1. ヒーロー: キャッチコピー+「1分診断をはじめる」CTA(最重要導線)
2. 探し方グリッド: 地図/好み/銘柄/料理/シーン(各カードに一言説明)
3. 今月の注目銘柄(手動キュレーション 4〜6件、購入ボタン付き)
4. 人気ランキング上位5件
5. 初心者向けガイド記事への導線
6. フッター: about / disclosure / privacy / 「20歳未満の飲酒は法律で禁じられています」

meta description は「1分の診断であなたに合う日本酒が見つかる。味わい・産地・料理から探せて、そのまま購入できる日本酒カタログ」等、ユーザーベネフィット中心に書き換える。

## 8. コンテンツ(SEO記事)`/guide/`

物販キーワードは競合が強いため、ロングテール記事で面を取る。初期10本の例:

- 日本酒の甘口・辛口とは?日本酒度の読み方
- 純米・吟醸・大吟醸の違い(図解)
- 初心者におすすめの飲みやすい日本酒10選
- 刺身に合う日本酒の選び方
- 日本酒の正しい保存方法
- 熱燗に向く日本酒とは
- プレゼントに喜ばれる日本酒の選び方(価格帯別)

各記事の末尾・文中から関連する銘柄詳細ページへ内部リンクする(記事→銘柄→購入 の流れ)。

## 9. データ設計

初期は `data/sakes.json`(または CSV → ビルド時変換)で開始し、100銘柄を超えたら Supabase 等へ移行。

```ts
type Sake = {
  id: string; slug: string;
  name: string; nameKana: string;
  breweryId: string; prefecture: string;
  classification: '純米大吟醸' | '純米吟醸' | '純米' | '大吟醸' | '吟醸' | '本醸造' | '普通酒';
  flavorType: 'kaori' | 'sou' | 'jun' | 'juku';
  taste: { sweetness: number; acidity: number; umami: number; aroma: number; sharpness: number }; // 1-5
  sweetDry: number;      // -3(甘) ~ +3(辛)
  lightRich: number;     // -3(淡麗) ~ +3(濃醇)
  specs: { polishing?: number; rice?: string; yeast?: string; abv?: number; smv?: number; acid?: number };
  servingTemp: ('雪冷え'|'花冷え'|'涼冷え'|'常温'|'ぬる燗'|'熱燗')[];
  pairings: string[];    // pairing category ids
  scenes: string[];      // scene ids
  priceRange: 1|2|3|4;
  description: string;   // オリジナル文
  affiliate: { rakuten?: string; amazon?: string; yahoo?: string }[]; // 容量別
};
```

- 初期投入は主要銘柄100本を目標(各都道府県最低1本)。味データは公式スペック+テイスティングノートを基に自前で採点する。

## 10. 法令・表記対応(必須)

1. **ステマ規制(景品表示法)**: 全ページ共通で「本サイトはアフィリエイト広告を含みます」を明示。`/disclosure` に詳細ポリシー。
2. **20歳未満飲酒防止**: フッター常設表記+初回訪問時の年齢確認モーダル(「20歳以上ですか?」、回答をlocalStorage等に保存)。
3. **プライバシーポリシー**: GA4等の計測利用を明記。
4. 特定商取引法表記は不要(自ら販売しないため)。
5. 各モールの規約遵守(Amazonは価格の静的表示禁止など。価格を出す場合はAPI経由か「参考価格」を出さない設計に)。

## 11. 技術要件

- Next.js App Router / `generateStaticParams` によるSSG+ISR(revalidate: 1日)
- 動的OG画像(`next/og`)
- `sitemap.xml` `robots.txt` 自動生成(`next-sitemap` または App Router組込み)
- Vercel Analytics + GA4
- Lighthouse モバイルで Performance / SEO 90以上を合格ライン
- モーダルUIは維持してよいが、開いた状態でURLが変わる実装(Intercepting Routes / Parallel Routes)にし、直リンクではフルページ表示にする
