## 1. SEO戦略 — 検索とAI検索の両取り

### 1-1. キーワードの3層構造

| 層 | 例 | 対応ページ | 特徴 |
|---|---|---|---|
| 指名系 | 「獺祭 味」「十四代 どこで買える」 | /sake/[slug] | CVR最高。銘柄数=戦線の広さ |
| 比較・選択系 | 「フルーティ 日本酒 おすすめ」「甘口 日本酒 初心者」 | /type/ /scene/ /ranking /guide | 検索量が多く収益性も高い激戦区 |
| 情報系ロングテール | 「日本酒度 マイナス 甘い」「ひやおろし いつ」 | /guide/ | 競合が弱く、初期に順位が取りやすい |

**序盤の定石は「下から攻める」**: 情報系で面を取り、内部リンクで比較系・指名系ページへ評価を流す(トピッククラスター構造)。ガイド記事は必ず「ハブ記事(例: 日本酒の選び方大全)→子記事→関連銘柄ページ」の3層リンクにする。

キーワード調査はラッコキーワード(無料枠)+Google Search Console。**公開2〜3ヶ月後のSearch Consoleが最良のネタ帳**になる: 「表示回数が多いのに順位11〜30位」のクエリを毎月抽出し、そのクエリ専用の見出し・記事を追加する。この改善ループが個人サイトSEOの本体。

### 1-2. 技術SEO(指示書§11への上乗せ)

- 構造化データの拡張: Product に加え、ガイド記事へ FAQPage、一覧ページへ ItemList、診断へ Quiz を実装。リッチリザルト獲得でCTRを上げる
- 内部リンクの自動化: 銘柄ページ下部の「類似銘柄」「同じ蔵の酒」「同県の酒」は、クローラビリティと回遊の両方に効く。孤立ページ(被内部リンク0)を検出するスクリプトをCIに入れる
- Core Web Vitals: 商品画像(楽天API由来)は next/image で遅延読み込み+サイズ指定。LCP対象のヒーローは静的アセットに
- 重複対策: 720ml/1800mlをページ分割しない(canonical1本)。/type/と/searchの絞り込み結果はnoindexで整理

### 1-3. AI検索(LLMO)対策 — 2026年の必須項目

ChatGPT・Claude・GoogleのAI Overviewが「おすすめの日本酒」系クエリの回答を代替しつつあり、**AIに引用されるサイト**になることが新しいSEO。

- 各ページ冒頭に「結論の1段落」(銘柄名+味の要約+こんな人向け)を置く。AIは冒頭の定義文を引用しやすい
- 事実をテーブル・箇条書きで構造化(スペック表は既に指示書に含む)
- サイトルートに llms.txt を置き、サイト構成と主要ページを記述
- 独自データを持つ: 「診断結果の統計(いちばん多い味覚タイプは◯◯)」「都道府県別の味傾向分析」のような**一次情報はAIにも人にも引用される**。さけのわデータの加工統計はこの用途に最適(帰属表示は維持)

### 1-4. E-E-A-T(信頼性)

- /about に運営者プロフィールを実名またはハンドル+顔出しに準ずる情報で。「なぜこのサイトを作ったか」を書く
- 可能なら実飲レビューを月数本追加(Experience)。全銘柄でなくてよい。「実際に飲んだ」印付き銘柄が混ざるだけで質が変わる
- 中期的には唎酒師などの資格取得が記事の権威性・プロフィールに効く(費用対効果は要判断)

## 2. コンテンツ拡張 — 検索需要の「山」を取りに行く

### 2-1. 季節カレンダー運用

日本酒の検索需要は季節イベントで大きく波打つ。**山の6〜8週間前に記事を公開**しておく(インデックスと順位上昇のリードタイム)。

| 時期 | 需要 | 用意するページ |
|---|---|---|
| 11〜12月 | お歳暮・年末年始・熱燗 | 贈答ガイド、燗酒特集 |
| 12〜3月 | 新酒・しぼりたて・生酒 | 新酒特集 |
| 5〜6月 | 父の日(年間最大級のギフト需要) | 父の日ギフト特集+価格帯別 |
| 9〜10月 | ひやおろし・秋あがり | ひやおろし特集 |
| 通年 | 誕生日・結婚祝い | 祝い酒ガイド |

ギフト系は客単価が高くアフィリエイト報酬も伸びるため、**/scene/gift は最優先で作り込む**。

### 2-2. UGC(ユーザーレビュー)の導入(Phase 3)

- 銘柄ページに星+一言レビュー投稿(ログイン不要、ニックネームのみ)
- 効果: ①ページの更新頻度と独自テキストが増えSEOに効く ②「まずい」「甘すぎ」等の率直な語彙がロングテールを拾う ③E-E-A-TのE
- リスク管理: 公開前モデレーション必須(スパム・誹謗中傷・未成年を示唆する投稿の排除)。Claude APIで一次審査→人間承認のフローにする

### 2-3. 英語版(Phase 4・将来)

海外の日本酒市場・訪日観光の文脈で「sake recommendation」系の英語検索は競合が非常に薄い。データ構造は流用できるため、紹介文の英訳生成+hreflang対応だけで参入可能。国内SEOが軌道に乗ってからの飛び地として有望。

## 3. SNS・渉外戦略 — 被リンクと指名流入を作る

### 3-1. 診断シェアをバイラルの軸に(最重要・低コスト)

性格診断系コンテンツがSNSで拡散する構造をそのまま使う。

- 診断結果ページに**タイプ名+味覚チャートを描いた動的OG画像**(next/ogで実装済み想定)と「結果をXでシェア」ボタン
- タイプ名はシェアしたくなる命名にする(例: 「華やか系・うたかたのロマンチスト」のようなキャラ付け)。12タイプの命名はClaudeでブレストし、人間が選定
- シェア文面テンプレート: 「私の日本酒タイプは【◯◯】でした。おすすめは△△ #日本酒診断」
- これが機能すると広告費ゼロで最も再現性の高い流入源になる

### 3-2. X(旧Twitter)運用

- 日本酒界隈はXが最も活発。「今日の一本」形式で銘柄ページの内容を毎日1ポスト(生成はClaudeでバッチ化、投稿は予約ツール)
- 10月1日「日本酒の日」、新酒解禁などのモーメントに診断を再プッシュ
- 酒蔵・酒販店アカウントへの引用や返信で界隈に認知を作る(宣伝臭を出さず、データや診断で貢献する立ち位置)

### 3-3. 酒蔵・業界への渉外(被リンク獲得)

個人サイトのドメイン評価を上げる王道は自然な被リンク。日本酒はリンク元候補が豊富:

1. **掲載連絡**: 銘柄を掲載したら蔵元に「貴蔵の◯◯を紹介しました」とメールで連絡。一定割合が公式サイト・SNSで紹介してくれる(=良質な被リンク)。連絡文テンプレートをClaudeで作成し、月10通ペース
2. **リンクされる資産を作る**: 「47都道府県 日本酒の味わいマップ」「日本酒フレーバー統計レポート」のようなデータビジュアルは、ブログ・ニュースサイト・卒論などから引用されやすい
3. **プレスリリース**: 診断リリース時・統計レポート公開時にPR TIMES(スタートアップ/個人向けプランあり)。業界メディア(SAKETIMESなど)への持ち込みも
4. 日本酒イベント(にいがた酒の陣等)のレポート記事は、指名検索の波に乗れて出展者からもリンクされやすい

### 3-4. Instagram / TikTok / YouTube

写真・動画は工数対効果が個人運営では重い。**優先度は X > Instagram >> 動画**。Instagramをやるなら診断結果カードとランキング画像の転載程度に留め、動画は売上が立ってから検討(または蔵元コラボ時のみ)。

## 4. 収益の多層化 — アフィリエイトの先へ

### 4-1. 既存アフィリエイトの最適化

- **ふるさと納税を追加する(即効性が高い見落とし枠)**: 楽天ふるさと納税・ふるなび・さとふる等はアフィリエイト案件があり、日本酒返礼品は寄付額が高額なため報酬単価が物販の数倍になる。銘柄ページに「ふるさと納税でもらう」ボタンを4つ目の導線として追加し、/guide/に「ふるさと納税でもらえる日本酒」特集を作る(検索需要も大きい)
- ボタン配置・文言・順序のA/Bテスト(Vercelのabテスト機能 or 単純な期間比較)
- 楽天スーパーセール・お買い物マラソン期間はランキングページを前面に出す(報酬発生が跳ねる時期)
- Amazon: 手動リンクの20銘柄を人気上位に置き、180日3件の適格販売を早期達成→PA-API解禁でリンク網羅率を上げる

### 4-2. 中期の収益源(月間数万PV到達後)

| 収益源 | 内容 | 目安時期 |
|---|---|---|
| 高単価ASP案件の面展開 | サブスク(saketaku等)・オンライン酒販の比較記事 | 6ヶ月〜 |
| Googleアドセンス/広告枠 | ガイド記事のみに限定(銘柄ページはCVR低下するので貼らない) | 6ヶ月〜 |
| タイアップ・純広告 | 蔵元・酒販店の新商品紹介記事(必ずPR表記) | 12ヶ月〜 |
| LINE公式/メルマガ | 診断完了者に「あなたのタイプの新着銘柄」を配信し再訪を作る | 12ヶ月〜 |

自社EC・在庫を持つ物販は行わない(酒類販売業免許・資金・物流が必要で、メディア運営の利点が消える)。

### 4-3. 収益方程式で考える

売上 = セッション数 × 購入ボタンCTR × モール側CVR × 単価 × 料率

- 序盤(〜6ヶ月)はセッション数(SEO・診断シェア)が支配的
- 中盤はCTR改善(ボタン配置・価格比較・ふるさと納税追加)
- 終盤は単価と料率(ギフト特化・高単価ASP・タイアップ)
と、フェーズごとに効かせるレバーを変える。

## 5. システム面の発展ロードマップ

### 5-1. 差別化機能: AI酒ソムリエ(Phase 3の目玉)

Claude APIを使った対話型レコメンド。「明日すき焼きなんだけど3,000円以内で甘めのやつ」→ サイト内の銘柄データ(フレーバー値+スペック+アフィリエイトリンク)をコンテキストに渡して回答させる。

- 実装: 銘柄データをそのままシステムプロンプトに入れる(100〜300銘柄ならRAG不要、全件注入で足りる)。回答には必ず銘柄ページへのリンクを含めさせる
- コスト制御: 1日あたりの利用回数制限+キャッシュ。月数千円以内に収まる設計に
- これ自体が「AIが日本酒を選んでくれるサイト」としてSNS・プレスのネタになる

### 5-2. パーソナライズと再訪設計

- 診断結果をlocalStorageに保存し、再訪時にトップを「あなたはこのタイプ」仕様に出し分け
- 「飲んだ・気になる」ボタン(ログイン不要)→ マイリスト → リストからの購入導線
- 将来ログイン導入時はSupabase Authで最小構成

### 5-3. 計測・実験基盤

- GA4(affiliate_clickほか診断完了・シェア・リスト追加をイベント化)+ Search Console + 楽天/各ASPレポートを**Looker Studioの1枚ダッシュボード**に集約。週1で見る数字を「セッション/クリック率/発生報酬/上位クエリ」の4つに絞る
- クリックログ(/api/out/)はVercel KVまたはSupabaseに蓄積し、「導線別・銘柄別の稼ぎ」を可視化 → 追加銘柄の選定に還流

### 5-4. 12ヶ月ロードマップ(目安)

| 期間 | 施策 | KPI目安 |
|---|---|---|
| 0〜2ヶ月 | Phase 1(銘柄ページ化・購入導線・法務)+Search Console登録 | インデックス100ページ |
| 2〜4ヶ月 | ガイド記事20本・季節特集・診断シェア実装・蔵元への掲載連絡開始 | 月間3,000セッション |
| 4〜6ヶ月 | ふるさと納税導線・UGCレビュー・統計レポート公開(PR TIMES) | 月間1万セッション・初報酬の安定化 |
| 6〜12ヶ月 | AIソムリエ・LINE配信・タイアップ営業・(検討)英語版 | 月間3〜5万セッション・運用費の黒字化 |

数値はあくまで目安。実際はSearch Consoleの伸びを見て前後する。
