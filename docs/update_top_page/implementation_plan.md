# トップページのリニューアルおよび都道府県選択機能の移植計画

本計画は、トップページのデザインを新しく作成された「雫 SAKE SELECT」の和風・高級感あるデザインに刷新し、同時にWebGLによる水面シミュレーションとWeb Audio APIによる水滴音の演出を導入します。また、既存のNext.jsアプリに備わっている「都道府県から選ぶ（インタラクティブな日本地図）」機能および「日本酒データベース」を、新しいトップページの構造とシームレスに統合します。

---

## ユーザー確認が必要な事項

> [!IMPORTANT]
> **1. スクロール挙動の制御について**
> 新しいデザイン（`sake/index.html`）では、画面全体が `100vh` 内に収まり、スクロールが制限されるスタイリング（`body { overflow: hidden; }`）となっています。しかし、Next.jsアプリの「日本酒一覧（`step === 'list'`）」や「お気に入り一覧」では、データ件数が多い場合に縦スクロールが必要になります。
> 本計画では、トップページ（`home`）、地方・都道府県選択（`region`）、診断（`quiz`）の段階では画面を固定（スクロール不可）とし、日本酒一覧（`list`）表示時にはスクロールができるように、表示状態に応じた動的なスクロール制御を導入します。
>
> **2. 味わいフィルターと既存データのマッピング**
> 「好みから探す」パネルにおいて、元のデザインでは「薫酒・爽酒・醇酒・熟酒」の4分類が定義されています。既存の日本酒データ（`Sake` 型）には `dry_sweet_index` (甘口・辛口) と `type` ('pure_rice' | 'ginjo' | 'daiginjo' | 'honjozo') が存在します。
> これらを以下のようにマッピングして、クリック時に自動で日本酒リストを絞り込んで表示します：
> - **薫酒** (フルーティー・大吟醸系) $\rightarrow$ `type === 'daiginjo' || type === 'ginjo'`
> - **爽酒** (すっきり・辛口系) $\rightarrow$ `dry_sweet_index < -2` (または `type === 'honjozo'`)
> - **醇酒** (芳醇・純米系) $\rightarrow$ `type === 'pure_rice'`
> - **熟酒** (濃厚・古酒系) $\rightarrow$ 今回のデータベースに対応する項目が少ないため、バランス型や特定条件で補完

## 提案する変更

変更は大きく分けて「スタイルの統合」「WebGL/音響コンポーネントの作成」「メインページ（`app/page.tsx`）の書き換え」の3つのステップで行います。

### 1. スタイル・フォントの定義

#### [MODIFY] [layout.tsx](file:///c:/Users/k-nis/Desktop/ネタBOX/日本酒サイト/app/layout.tsx)
- Google Fonts から `Shippori Mincho B1`, `Cormorant Garamond`, `Zen Kaku Gothic New` を `next/font/google` で読み込むように定義し、CSS変数 `--font-display`, `--font-latin`, `--font-body` を割り当てます。

#### [MODIFY] [globals.css](file:///c:/Users/k-nis/Desktop/ネタBOX/日本酒サイト/app/globals.css)
- `sake/assets/css/style.css` のスタイル定義（CSS変数、フォント割り当て、キャンバス、ノイズ、ヘッダー、パネル、各種カードデザイン、トランジションなど）を移植します。
- 既存の TailwindCSS やスクロールバーのカスタマイズと衝突しないように調整します。
- `body` に適用されている `overflow: hidden` を、Next.js 用に `body.fixed-screen` などの特定のクラスに限定し、日本酒リスト表示時などにはスクロール可能にします。

### 2. 水面・音響演出の実装

#### [NEW] [WaterBackground.tsx](file:///c:/Users/k-nis/Desktop/ネタBOX/日本酒サイト/components/WaterBackground.tsx)
- WebGLによる3D水面描画と Canvas 2D による雫の落下シミュレーションを行う React コンポーネントを新規作成します。
- Web Audio API による水滴音の生成ロジック（`synthesis.js`）もこのコンポーネント内で初期化・制御します。
- ボタンホバー時の波紋トリガーなどを外部から実行できるように、`window` オブジェクトへの一時的な登録か、または React の参照（`useRef`）等を介して呼び出せる設計にします。

### 3. トップページロジックの書き換え

#### [MODIFY] [page.tsx](file:///c:/Users/k-nis/Desktop/ネタBOX/日本酒サイト/app/page.tsx)
- メインロジックをリファクタリングし、状態 `step` に応じてパネルの表示/非表示（`is-active` クラスによるトランジション制御）を行います。
- 水面トランジション（`overlay-transition`）のアニメーションを React のステートと組み合わせて実装し、画面遷移時に自動的に実行されるようにします。
- 各パネルの内容を以下のように実装します：
  - **`panel-map`（地図から探す）**: ユーザーの要望通り、既存の `JapanMap` と「都道府県リスト」をそのまま配置し、日本地図から選べるようにします。
  - **`panel-taste`（好みから探す）**: 4つのタイプ（薫・爽・醇・熟）のカードを表示し、クリックするとそのタイプに絞り込まれた `list` ステップへ遷移します。
  - **`panel-brand`（銘柄から探す）**: データベースから全銘柄を取得し、和風の縦スクロールリストとして表示します。
  - **`panel-diagnosis`（診断から探す）**: 既存の4つの質問（味、度数、香り、場面）に回答するUIを「雫」のデザインに合わせて実装し、結果表示からおすすめの日本酒へ遷移させます。
  - **`list`（日本酒一覧）** & **`favorites`（お気に入り）**: 新しいデザインの統一感を保った美しいカード形式にリフレッシュして表示します。

## 検証計画

### 自動検証
- 開発サーバー (`npm run dev`) を実行し、ビルドエラーや TypeScript の型エラーが発生しないことを確認します。

### 手動検証
- ブラウザでトップページを開き、WebGL水面の描画、波紋の発生、クリック時・ホバー時の水滴音（Web Audio API）が正常に機能することを確認します。
- 「地図から探す」をクリックし、トランジション演出を経て日本地図（`JapanMap`）が表示され、都道府県をクリックするとその地域の日本酒リストへ正しく遷移することを確認します。
- 「好みから探す」「診断から探す」「銘柄から探す」の各機能が、正しくデータベースのデータと紐づいて動作することを確認します。
- お気に入り機能の動作を確認します。
